## 2. 게임회사별 인재상 & 핵심가치

### 넥슨
- 인재상 키워드: (채용 페이지 접근 제한으로 상세 미확인)
- 넥슨게임즈 7가지 가치: 유저 중심, 용기, 효율성, 권한과 책임, 주인 의식/오너십

### 넷마블
- **인재상 3가지**:
  - Challenger(도전자): 열정, 도전, 조직력, 동료
  - No.1 Player: 일류, 큰 꿈, 자부심
  - Innovator(혁신가): 혁신, 글로벌, 즐거움
- **업무 핵심가치**: 디테일의 힘, 전략적 사고, 스피드
- 기업철학: "세상의 변화를 선도하며 최고를 추구"

### 크래프톤
- 꿈: "존경 받는 제작 리더십과 품격 있는 구성원들이 명작 게임을 만드는 것"
- 핵심가치: 넓게 보는 전문가, 유저 관점, 변화 적응+근성, 생산적 커뮤니케이션, 책임과 신뢰
- 글로벌 기준에서 바라보는 시각 중시

### 스마일게이트
- **인재상 3가지**:
  - 열정(Passion): 포기하지 않고 목표 능동적 달성
  - 사명감(Commitment): 역할과 의무에 보람, 끝까지 책임
  - 이해(Understanding): 경청 기반 소통, Mega Value 이해
- 채용 중점: 스펙보다 '열정', 가시적 성과를 낸 경험
- 자소서 팁: 소중한 가치 + 그 이유/계기 + 가치를 지키기 위한 행동/경험

### 엔씨소프트
- 핵심가치: Integrity(진지함), Passion(헌신), Never-ending Change(감동)

## 3. 게임업계 자소서에 쓰지 말아야 할 것 (클리셰/금기)

출처: brunch.co.kr/@sodrock/1 + 기타 수집

### 금기 1: 게임에 빠져서 인생 망친 이야기
- 게임사도 자기관리 못하는 사람 원하지 않음
- 게임 잘하는 것 ≠ 기획 능력
- 예외: "게임에 빠진 경험 → 직접 만들겠다는 결심 → 구체적 노력과 성과" 연결 시 OK

### 금기 2: 장편 성장소설
- 평범한 성장과정 길게 서술 → 마이너스
- 담당자가 끝까지 안 읽을 확률 높음
- 예외: 독특한 배경(오락실 사장 자녀, 외국 거주 등)이 업무와 연결될 때

### 금기 3: 근거 없는 자랑
- "매우 창의적이며 성실합니다" → 설득력 없음
- 구체적 사실 기반이어야 함 (변화를 만든 사례, 인정받은 경험 등)

### 금기 4: "게임을 좋아해서 지원했습니다"
- 가장 흔한 클리셰
- 게임을 좋아하는 것과 게임을 만드는 것은 완전히 다른 능력

### 금기 5: 비현실적 포부
- "입사 후 5년 내 매출 1위 게임 개발" 같은 표현

### 금기 6: 성격 설명 (책임감, 리더십 등 일반적 덕목)
- 구체적 사례 없이 성격만 나열하는 것

### 금기 7: AI가 쓴 티 나는 문장
- 인사담당자가 AI 활용 자소서 탐지 가능
- 본인의 진솔한 목소리가 중요

### 금기 8: 복붙 자소서
- 다른 회사에 낸 자소서 그대로 재활용
- 지원 회사에 대한 이해 부족이 드러남

## 4. 직군별 자소서 핵심 포인트

### 게임 기획자
- **필수 어필**: 기획자 지망 이유 + 직군별 관심/열정의 과정과 결과
- **포트폴리오 삼위일체**: 관심 있는 게임 = 지원 회사 게임 = 포트폴리오 게임
- **자소서 역할**: 능력 검증이 아닌 "궁금증 유발 → 포트폴리오/면접 연결"
- **피해야 할 것**: 성장 배경, 성격 설명, 경력 단순 나열, 비현실적 포부
- **포트폴리오**: 질 > 양, 2개 정도면 충분, 직군별 특화 (시스템/콘텐츠/UI 혼합 금지)
- **평가 기준**: 기획자적 사고 ("무엇을, 왜"), 기본 직무 지식

### 서버/클라이언트 프로그래머
- **자소서 필수 3요소**: 프로그래머 선택 이유 + 목표 + 지원 회사 선택 이유
- **강점 표현 3테마**: 지식(학교 학습) + 경험(공모전/활동) + 성격(프로그래머 업무 연결)
- **기술 스택**: 주요 개발 언어, 보조 언어, 개발환경, 사용 도구 구체적 작성
- **면접 준비**: C++, 알고리즘, 자료구조, OS, 그래픽스, 선형대수학
- **핵심 팁**: 모르면 솔직히 "모른다", 아는 것은 자세히 설명

### 아트/원화
- **포트폴리오가 자소서보다 중요** (면접 여부 = 포트폴리오로 결정)
- **회사 프로젝트에 맞는 스타일**: 아무리 잘 그려도 회사 스타일과 안 맞으면 탈락
- **다양성**: 다양한 나이/성별/종족 소화 능력 필요
- 캐릭터 삼면도, 컨셉 바리에이션, 스킬 컨셉, 몬스터 등 포함

### QA/테스터
- **지원동기**: 완성도 높은 게임 만들고 싶다는 열정 중심
- **직무역량**: 꼼꼼함, 문제 해결력, 협업 강조
- **경험**: 테스트 계획 수립, 버그 리포트 등 구체화
- **가점 요소**: 엑셀 함수/수식 능력, ISTQB/CSTS 자격증
- **면접 질문**: QA 직무 지원 이유, 게임 테스트 경험, 협업 경험, 스트레스 대응

### 마케팅/사업
- **핵심 역량**: 설득력, 통찰력, 네고시에이션
- **경험 어필**: "무언가를 효과적으로 팔아본 경험" (원룸 판매도 마케팅 경험)
- **특성**: 외근/미팅 잦음 → 사람 만나는 것을 좋아해야 함
- **중요**: 구체적 수치, 확인 가능한 자료 제시

## 5. 공통 자소서 작성 원칙

### DO (해야 할 것)
- 회사가 추구하는 방향 파악 (신년인사, 비전, 목표)
- 구체적 수치와 근거 제시
- 문단 짧게 나누고 대표 제목 넣기
- 지원 회사별 맞춤 작성
- 게임 플레이 경험 깊이 있게 분석
- 직무와 연결되는 구체적 에피소드

### DON'T (하지 말아야 할 것)
- 다른 회사 자소서 복붙
- 추상적 자기 PR ("창의적", "성실한")
- 게임 플레이 = 직무 능력으로 착각
- 장황한 성장 과정 서술
- AI로 작성한 티 나는 문장
- 포트폴리오 없이 자소서에 능력만 나열

### 구조
- "주제문 + 구체적 설명" 형식
- 지원동기 → 역량/경험 → 포부 순서
- 회사/직무 키워드를 자연스럽게 녹이기
